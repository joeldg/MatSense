# 🚀 Phase 10: Native Core Benchmarking (Mojo Sandbox)

This is an isolated experiment to evaluate the speed of rewriting the Grappling AI kinematic physics engines in Modular's **Mojo** language.

Mojo is a superset of Python that compiles down to hardware-specific AI architectures via LLVM/MLIR, offering C-level speeds (and hardware `SIMD` optimization) without sacrificing Pythonic readability. 

Unlike the Golang test (which used `ctypes` overhead to marshal memory from Python), this script runs entirely natively within the Mojo compiler as a standalone driver.

## 1. Installing the Modular SDK
To run the `.mojo` benchmark, your machine needs a compatible environment. Modular now uses **Pixi** for package management and environment setup.

You can install `pixi` and initialize the Mojo environment in your terminal:
```bash
# 1. Install Pixi (macOS/Linux)
curl -fsSL https://pixi.sh/install.sh | bash

# 2. Add Pixi to your path (or restart your terminal)
# source ~/.bashrc or ~/.zshrc

# 3. Create a new Pixi project with Modular channels
# You MUST include the Modular and conda-forge channels, or Pixi won't find the package!
pixi init -c conda-forge -c https://conda.modular.com/max
pixi add "mojo>=24.2"
```

## 2. Running the Benchmark
Inside the `.mojo` script, we statically defined the `calculate_fast_kuzushi` algorithm using strict `Float64` constraints to allow the LLVM compiler to aggressively optimize the physics math.

Once the SDK is installed and the Pixi environment is active, you can execute the 100,000-frame benchmark natively:
```bash
pixi run mojo benchmark_kuzushi.mojo
```

### Alternatively: Compile to a Standalone Executable
Mojo can compile Python-like scripts directly into raw binaries that require no runtimes or Python installations on the target machine:
```bash
mojo build benchmark_kuzushi.mojo -o mojo_kuzushi
./mojo_kuzushi
```

Compare the output of this run against the ~0.25s generated by `experiments/native_core/benchmark_native.py` (Golang).
